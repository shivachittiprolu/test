8.5. Which providers are approved for use on TFE?

Currently the use of the google and google-beta providers is supported on TFE. These providers allows for the GCP APIs to be used to allow for the creation of Google Cloud Resources.

8.6. Is there a roadmap of providers which will be available to use in the future?

There is currently no published roadmap for the upcoming expectations of new providers or there support.

There is however a declared intention to do so through 2021-05-14's update to the Terraform Enterprise Design &  Deployment CPDA (Decision Outcomes section). Once the target state has been rolled out this will be enabled the use of the kubernetes provider as an example.

8.7. How can providers be added to the approved list?

No new providers can be added to the approved list at the moment because they require network connectivity to the target non-gcp services which is not yet possible (see roadmap providers question above). In the meantime however, you can register your interest in using a particular un-supported provider in the LZ - General Support symphony channel. We can't guarantee its availability or approval, though.

8.8. A GCP resource I provision using TFE, should trigger an action on an on-prem system - what's the recommended way to do this?

Managing on-prem systems with DBTFE is not part of its roadmap. This is not possible.
